VEOCreate

The VEOCreate tool creates one or more VEOs from:

  • Source digital record(s)
  • Control file
  • Metadata templates
  • PFX file

Source digital records

These are the records to be converted into VEOs. VEOs can be constructed at an individual record level (one VEO for each record) or at multi record level (e.g. one VEO for a collection of records). The digital records should be arranged in the preferred way (single and/or multi-level) and should be in an approved long term preservation format prior to VEO creation.

Control file

The control file is a text file containing multiple rows of tab separated text commands. The control file instructs the VEOCreate tool on:

  • How the VEOs are to be constructed
  • What record(s) are in each VEO
  • What metadata is to be included in each VEO
  • What PFX file to use to sign the VEOs

Metadata templates

The metadata template files are used to generate the metadata packages for each VEO. They are text files expressed in XML which indicate:

  • Which metadata standard has been used (e.g. AGLS)
  • What syntax (i.e. rules/language to structure the metadata) have been applied (e.g. RDF)

PFX file

The PFX file is a unique certificate and private key that is used to ‘sign’ each VEO. A PFX file is protected against reading and requires a password. Government agencies are responsible for obtaining their PFX file for VEO creation.

VEOAnalysis

The VEOAnalysis tool allows users to examine and test VEOs. It performs any combination of the following tasks:

  • Unpacking VEO files into VEO directories
  • Testing VEOs for errors and questionable practices
  • Producing a visualisation of the contents of a VEO

VEOAnalysis may work on one VEO or multiple VEOs.

VEOSign

The VEOSign tool performs three tasks on existing VEOs:

  • Removes signature files
  • Re-signs the VEO(s)
  • Zips the contents of the VEO directory to create a new VEO

The intended use of VEOSign is to allow the modification of existing VEOs. For VEOs to be successfully re-signed they must firstbe unpacked using VEOAnalysis. The contents of the VEO can then be manually modified, and finally VEOSign invoked to regenerate the VEO.

In practice, VEOSign is very similar to VEOCreate.
